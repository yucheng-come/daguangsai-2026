作品三：「一切从问开始」AI创作工作流

1. 创意方法论：苏格拉底式提示词工程

使用工具：千问大模型

本作品的核心创新点在于——我们不是让AI给答案，而是让AI帮人问出更好的问题。这需要设计一套「苏格拉底式提示词链」，让千问按照既定的哲学深度递增路径，对用户的原始问题进行层层追问。

2. 三层问题进化Prompt设计

第一层：表层追问 — 揭示真正关切

这是一个好问题。让我们深入一层：这个问题背后真正的关切是什么？不要直接回答这个问题，而是帮提问者发现——他们真正在意的可能是什么。请将这个问题变得更本质一些，用一个新的、更深刻的问题来替代它。

第二层：假设检验 — 挑战前提

更深一层：这个问题的前提假设是什么？如果这些假设不成立——或者说，如果提问者关于这个问题的所有"理所当然"都是错的——那么问题本身会发生什么变化？请重新表述这个问题，让它剥离所有未经审视的假设。

第三层：视角转换 — 未来回望

换一个视角：想象你是5年后的提问者本人。那时你已经拥有了更多的经历和智慧。从那个未来的视角回望——你现在纠结的这个问题，会变成什么样？请不要直接回答原始问题，而是输出一个经过三层进化后的、接近问题本质的新问题。

3. 问题进化树可视化

每层进化后的新问题与原始问题之间形成「根→枝→叶」的树形结构。前端CSS绘制连接线和层级缩进，直观展示从表层到本质的思维深化过程。千问辅助生成了树形结构的HTML/CSS代码和动画效果。

4. 洞察生成

在展示完问题树之后，千问基于原始问题和三层进化结果，生成一段「洞察总结」——不是在回答问题，而是揭示提问者在这个过程中的思维变化，让用户感受到问题进化本身的价值。

提示词工程方法论总结

通过三件作品的实践，我们总结出以下AI创作方法论：

方法一：角色锚定法

为千问设定具体的「角色+经验+语气」，而非笼统地要求输出。例如「你是一个10年经验的资深后端架构师」比「请帮我分析bug」产生的输出质量高一个数量级。

方法二：链式追问法

不期望一次prompt得到最终结果。先将复杂任务拆解为多个子任务，每个子任务用独立的prompt，前一个输出作为后一个的输入。本报告中的三件作品均采用此方法。

方法三：多风格输出法

同一内容要求千问以多种风格输出，从中选择最佳版本或组合使用。作品一中的四种文学风格就是典型实践。

方法四：反向约束法

除了告诉AI「要什么」，更重要的是告诉AI「不要什么」。例如作品三中每一层都明确约束「不要直接回答原始问题」，迫使AI进行更深层的思考。

方法五：人工-AI协作边界

环节AI负责人类负责

文案内容生成初稿和多风格变体精修、校准品牌调性
